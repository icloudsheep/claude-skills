# code-comment

代码注释规范 skill。核心理念：**注释权责对齐当前作用域**——注释只描述当前方法 / 类 / 字段自身的逻辑，不向上溯源调用链、不向下探索消费方，从源头上避免注释随项目演进而腐化。

## 适用场景

写 / 改任何代码注释前触发，覆盖 javadoc、行内注释、字段注释、测试注释。

## 要点速览

- **不向上溯源**：不写「唯一入口」「都收敛于」「调用来源」「由上游 xx 保证」。
- **不向下探索**：不写「产物去向」「下游消费方」「对调用方无影响」。
- **不横向引用**：不写「与主链路 xx 保持一致」。
- **阶段性现状带时间戳**：会过期的现状用 `[注: 截至 YYYY-MM-DD]`（日粒度）。
- **不写行号**：指向他处用方法名 / 字段名引用。
- **不脑补业务**：不确定时只描述代码行为。
- **鼓励补「为什么」**：取舍、边界坑点、复杂逻辑意图主动补注释。
